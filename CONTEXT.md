# Weave Domain Glossary

## Core Concepts

### Task Lifecycle

- **Job**: User-submitted task/requirement. Has status lifecycle (QUEUED → LEASED → RUNNING → SUCCEEDED/FAILED). Persisted in `control_plane/repository.py`. A Job may have multiple Runs.
- **Run**: Single execution attempt of a Job. Contains a DAG execution. A new Run is created on each retry.
- **DAG**: Directed Acyclic Graph. The execution plan generated by the Orchestrator from a requirement. Contains DAGNodes and DAGEdges.
- **DAGNode**: A single step within a DAG. Has agent_type, prompt, dependencies. Executed by an AgentBackend.
- **Requirement**: Natural language description of what the user wants done. Input to `Orchestrator.plan()`.

### Agent Execution

- **AgentBackend**: Abstract interface for executing a DAGNode (`agent/backends/base.py`). Implementations: BuiltinBackend, ClaudeCodeBackend, CodexBackend.
- **WorkerAgent**: An agent instance managed by AgentPool. Wraps the built-in agent loop (LLM call + tool use cycle).
- **Orchestrator**: LLM-driven planner. `IntelligentOrchestrator.plan()` takes a requirement and produces a DAG. Uses Opus for planning (M5 decision).

### Workspace Isolation

- **ExecutionBackend**: Abstract interface for workspace isolation (`backend/base.py`). Implementations: LocalBackend, WorktreeBackend. Manages filesystem, NOT agent execution.
- **Worktree**: Git worktree providing filesystem isolation per Job/Run. Managed by WorktreeBackend.

### Observability

- **Event**: Structured occurrence recorded in session store (EventType enum). Append-only JSONL. Covers workflow, agent, node, and system events.
- **Trace** *(M5 new)*: Hierarchical record of a complete DAG execution. Composed of Spans. Stored as JSONL events, optionally exported via OTLP.
- **Span** *(M5 new)*: Unit of work within a Trace. 4 layers: Run → Node → LLM Turn → Tool Call. Follows OpenTelemetry conventions.

### GitHub Integration *(M5 new)*

- **Issue Poll**: CLI-driven mechanism to fetch GitHub Issues with `weave` label and execute them.
- **Issue Pipeline**: Issue → DAG → Execute → PR. The end-to-end automated workflow.
- **Branch Strategy**: `fix/{issue-number}-{slug}` per Issue. Force push on retry.
- **Label Lifecycle**: `weave` → `weave-running` → `weave-pr` / `weave-failed`

### Token Tracking

- **BudgetConfig**: Per-run token budget configuration. Managed by BudgetManager.
- **Token Report** *(M5 new)*: Per-Job token summary (aggregated across all Runs). Per-Run trace records token counts at each layer.
